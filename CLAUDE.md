# bughunterv2 — 项目规则

> 本文件每个会话自动加载，是必须遵守的核心规则。完整文档见 `docs/README.md`。

## 北极星（项目目标）

用 agent 对一个 Java + Maven + SVN 目标项目做线上 bug 的**根因诊断与人在环修复**——把它做出来、做好、做出价值。

## 协作铁律（详见 `docs/standards/collaboration.md`）

1. **渐进式搭建**：工程庞大，一步一步充分沟通，不试图一条消息解决；信息不足先问，不硬猜。
2. **有主见、敢异议**：不盲从；发现问题 / 风险 / 更优解，主动指出并讨论后再做。
3. **锚定目标、防偏航**：对齐北极星，发现偏离时主动 `⚠️ warn` 并反驳、给替代意见。
4. **互相提醒**：老板首次做此类项目，agent 主动预警风险与遗漏。

## 工作纪律

- 设计先行，**不过早编码、不过度设计**。
- 需老板拍板时，优先用 **AskUserQuestion 点选（带推荐项）**，降低输入成本（老板公司内无外网、手机输入慢）。
- 决策记 `docs/adr/`，概念记 `docs/glossary.md`，设计记 `docs/design-docs/`。
- 敏感操作（写库、调真实接口、SVN 提交）必须**人工 gate**。
- 上下文控制：大数据不整体进 context，带假设查证 + 最小切片 + 落盘引用。
